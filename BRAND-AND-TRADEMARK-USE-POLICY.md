## Why this file is short

The canonical `BRAND-AND-TRADEMARK-USE-POLICY.md` for the UniCORE-family programmes lives in the **TrueAI Foundation** repository and is mirrored across the foundation triad (UniVERSE / TrueAI / UniCORE-AI) for consistent reading. The full text — including the badge-denial-on-military structural rule (§4.5) and the two-mechanism structure (§8: CC BY 4.0 governs the work, certification mark governs the badge) — is canonical there.

This file in the **UniCORE.GVB** repository exists to:

1. Make the policy discoverable from the UniCORE.GVB name surface.
2. Confirm that the same rule applies to the `UniCORE.GVB` name and to the open list of `UniCORE.GVB.<Industry>` classified names (the substrate-services layer's equivalent to the `UniCORE.<Industry>` classification list).
3. Cross-link to the canonical version.

## Scope extension to UniCORE.GVB

For the avoidance of doubt, the canonical policy applies in full to:

- The `UniCORE.GVB` name itself.
- The open list of `UniCORE.GVB.<Industry>` classified names (`UniCORE.GVB.Law`, `UniCORE.GVB.Healthcare`, `UniCORE.GVB.Banking`, `UniCORE.GVB.Accounting`, `UniCORE.GVB.Space-Industry`, etc., and any future entrants on the list).
- The `Powered by UniCORE AI` and `Built on the TrueAI Foundation` attribution badges, as applied to a substrate-services-layer deployment that bears a `UniCORE.GVB.<Industry>` name.

§4.5 (badge-denial-on-military-uses) and §8 (two-mechanism structure: CC BY 4.0 governs the work, certification mark governs the badge) apply identically here.

## Substrate-services-layer specific note

UniCORE.GVB is the **substrate-services layer** of the programme. It provides production-grade infrastructure (mail, file transfer, DNS, federation, tenancy, topology, FTP, Outlook integration, web disk, calendar/contacts, server profiles, node operations, bandwidth and storage reporting, jurisdictional posture, inter-node bridging) that any Vertical Solution may run on top of.

Because UniCORE.GVB sits below the Vertical Solution layer, the badge-denial rule has a sharper edge here than at the Vertical layer: a Vertical Solution can be built badge-free; a substrate that powers an explicitly military Vertical cannot escape the rule by claiming it is "just substrate." The classification of the workloads running on the substrate is the controlling fact.

## When industry-classified sibling repos are created

When `UniCORE.GVB.Law`, `UniCORE.GVB.Healthcare`, etc. are created as their own public repositories (see `ROADMAP.md` and the certification-trigger lock 2026-05-21 12:33 UTC), each will carry its own `BRAND-AND-TRADEMARK-USE-POLICY.md` — initially as a cross-reference of this same shape, and graduated to a fuller per-classification text only if and when programme reasons require.
